# Phase 1 테스트 케이스 요약

- **총 케이스 수**: 17개
- **섹션 수**: 5개

## 섹션 구조

### 1. Phase 1 - 빌드 라벨 테스트 (3개 케이스)
- [TC-P1-001] 런 생성 시 빌드 라벨 입력
- [TC-P1-002] 빌드 라벨 없이 런 생성
- [TC-P1-003] CSV Export에 빌드 라벨 포함 확인

### 2. Phase 1 - 버그 링크 테스트 (3개 케이스)
- [TC-P1-004] 결과 입력 시 버그 링크 추가
- [TC-P1-005] 여러 개의 버그 링크 입력
- [TC-P1-006] CSV Export에 버그 링크 포함 확인

### 3. Phase 1 - 케이스 버전 테스트 (3개 케이스)
- [TC-P1-007] 새 케이스 생성 시 버전 1로 초기화
- [TC-P1-008] 케이스 내용 수정 시 버전 증가
- [TC-P1-009] 우선순위/태그 변경 시 버전 미증가

### 4. Phase 1 - 런 스냅샷 테스트 (3개 케이스)
- [TC-P1-010] 런 생성 시 케이스 스냅샷 저장
- [TC-P1-011] 케이스 버전 스냅샷 저장 확인
- [TC-P1-012] 과거 런에서 케이스 추적성 확인

### 5. Phase 1 - CSV Export 테스트 (5개 케이스)
- [TC-P1-013] CSV Export 버튼 표시 및 동작
- [TC-P1-014] CSV 파일 컬럼 구조 확인
- [TC-P1-015] Excel에서 한글 깨짐 없이 열기
- [TC-P1-016] NotRun 상태 케이스 CSV Export
- [TC-P1-017] 섹션 경로 표시 확인

## 테스트 실행 방법

### 1. 프로젝트 2 접속
```
http://127.0.0.1:5000/p/2/cases
```

### 2. 섹션별로 테스트 진행
각 섹션의 테스트 케이스를 순서대로 실행하면서 Phase 1 기능을 검증합니다.

### 3. 런 생성 예시
```
런 이름: Phase 1 Validation Test
빌드 라벨: v1.0.0-phase1
타입: Regression
케이스: Phase 1 관련 케이스 전체 선택
```

### 4. 검증 포인트

#### 빌드 라벨
- ✅ 런 생성 폼에 빌드 라벨 입력 필드 존재
- ✅ 런 목록에 빌드 라벨 표시 (🏷️ 아이콘)
- ✅ 런 실행 페이지 상단에 빌드 라벨 표시
- ✅ CSV Export에 Build Label 컬럼 포함

#### 버그 링크
- ✅ 결과 입력 UI에 버그 링크 입력 필드 존재
- ✅ 여러 개의 버그 링크 입력 가능 (쉼표 구분)
- ✅ 결과 히스토리에 버그 링크 표시
- ✅ CSV Export에 Bug Links 컬럼 포함

#### 케이스 버전
- ✅ 새 케이스 생성 시 version=1
- ✅ 제목/단계/예상결과 수정 시 버전 증가
- ✅ 우선순위/태그 변경 시 버전 미증가

#### 런 스냅샷
- ✅ 런 생성 시 케이스 버전 저장
- ✅ 런 생성 시 케이스 내용 스냅샷 저장
- ✅ 케이스 수정 후에도 과거 런은 이전 내용 유지

#### CSV Export
- ✅ 런 실행 페이지에 CSV Export 버튼 존재
- ✅ 클릭 시 CSV 파일 다운로드
- ✅ 모든 필수 컬럼 포함
- ✅ Excel에서 한글 정상 표시 (UTF-8 BOM)
- ✅ NotRun 상태 케이스 처리
- ✅ 섹션 경로 표시
